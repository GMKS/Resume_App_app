## B) Critical Issues (Blockers)

### 1. Live Twilio secrets are committed and packaged
- Files:
  - [c:\Resume_App_app\.env](c:\Resume_App_app\.env#L2)
  - [c:\Resume_App_app\pubspec.yaml](c:\Resume_App_app\pubspec.yaml#L77)
  - [c:\Resume_App_app\build\flutter_assets\.env](c:\Resume_App_app\build\flutter_assets\.env#L2)
- Why this blocks release:
  - The app currently ships long-lived Twilio credentials to every installed client.
  - This allows OTP abuse, account billing abuse, and full compromise of the Verify service.
  - This is a serious user-data and platform-security failure.
- Minimal safe fix:
```text
1. Remove `.env` from Flutter assets.
2. Rotate the exposed Twilio Account SID/Auth Token/Verify SID immediately.
3. Move OTP send/verify to a server-side endpoint or Firebase Callable Function.
4. Keep only a server URL in the app, never Twilio secrets.
```

### 2. Twilio Basic Auth is constructed on-device
- File: [c:\Resume_App_app\lib\core\services\twilio_service.dart](c:\Resume_App_app\lib\core\services\twilio_service.dart#L56)
- Why this blocks release:
  - Even if `.env` were removed from assets, any client-side Twilio auth strategy is structurally unsafe because the app must contain the credential to call Twilio directly.
- Minimal safe fix:
```dart
// Client
final response = await http.post(
  Uri.parse('$backendBaseUrl/auth/send-otp'),
  body: {'phone': phoneNumber},
);

// Server holds Twilio credentials and calls Twilio Verify.
```

### 3. Free export watermark is not enforced in the exported file
- Files:
  - [c:\Resume_App_app\lib\features\preview\screens\preview_screen.dart](c:\Resume_App_app\lib\features\preview\screens\preview_screen.dart#L644)
  - [c:\Resume_App_app\lib\features\preview\services\preview_pdf_service.dart](c:\Resume_App_app\lib\features\preview\services\preview_pdf_service.dart#L32)
- Why this blocks release readiness:
  - The premium promise says free users get watermarked exports, but the watermark currently exists only as an on-screen overlay.
  - This is a monetization correctness issue and can undermine listing claims and support disputes.
- Minimal safe fix:
```text
Add a watermark layer in the PDF-generation path when `FreePlanService.shouldShowWatermark` is true, instead of only in the preview widget tree.
```

### 4. AI API keys are stored unencrypted
- Files:
  - [c:\Resume_App_app\lib\features\settings\screens\settings_screen.dart](c:\Resume_App_app\lib\features\settings\screens\settings_screen.dart#L1557)
  - [c:\Resume_App_app\lib\core\services\ai_resume_service.dart](c:\Resume_App_app\lib\core\services\ai_resume_service.dart#L1738)
- Why this is a blocker or near-blocker:
  - Users’ own paid AI credentials are persisted in plaintext and reused across multiple AI screens.
  - This is high-risk storage for sensitive tokens.
- Minimal safe fix:
```text
Move AI API key persistence to `flutter_secure_storage` and migrate reads/writes through a single secure-key service.
```

### 5. Weak cloud-sync isolation for sensitive resume data
- File: [c:\Resume_App_app\lib\core\services\supabase_sync_service.dart](c:\Resume_App_app\lib\core\services\supabase_sync_service.dart#L56)
- Why this is a blocker or near-blocker:
  - Sensitive resume content can be keyed by a short user-selected sync code and anonymous auth.
  - This is weak access control for PII-heavy data.
- Minimal safe fix:
```text
Keep manual backup UX, but bind cloud records to authenticated user identity or a cryptographically strong device-bound secret rather than a simple shared code.
```

## J) Final Action Plan

1. Immediate containment: rotate the exposed Twilio credentials and purge `.env` from shipped assets and future builds.
2. Structural auth fix: replace direct Twilio API calls with a backend or callable-function proxy.
3. Secure storage: move AI API keys and any other sensitive tokens from SharedPreferences to secure storage.
4. Cloud data hardening: replace anonymous-auth plus sync-code sharing with authenticated ownership or a stronger encrypted sharing mechanism.
5. Freemium correctness: enforce watermarking inside the generated PDF output, not just the preview UI.
6. Metadata cleanup: replace or remove placeholder Facebook and iOS auth configuration.
7. Data-safety alignment: update in-app disclosure copy and your Play Data Safety form to reflect OTP, cloud backup, AI processing, and social sign-in data handling accurately.
8. Cleanup: remove stale docs that instruct hardcoding secrets and resolve unused-private-member warnings in template and preview families.
9. Regression pass: rerun focused tests for OTP, backup/restore, free export watermarking, and subscription entitlement once the above changes are made.
